# Foundation-Model-Guided UAV Navigation — Notes

## Goal

Build a system where a vision-language model replaces the classical frontier planner for indoor UAV exploration, and check whether it actually explores better than the geometric baseline in AirSim.

See [the design spec](../specs/2026-06-15-skymind-design.md) for the full problem statement and related work.

---

## Things to Figure Out / Learn Before Building

- ROS 2 nodes and topics — how the pieces (camera, map, planner, flight controller) will talk to each other
- Running a vision-language model locally (ollama + LLaVA / LLaMA-Vision) and getting clean, structured output back instead of rambling prose
- Turning a depth camera feed into a 2D occupancy grid (explored / unexplored / obstacle)
- MAVROS OFFBOARD mode — how to actually send a drone a "fly to this point" command safely
- How to design a fair A/B comparison (VLM planner vs geometric planner, same everything else) so any difference in results is real

## Reference Papers

Same list as in the [design spec](../specs/2026-06-15-skymind-design.md):
- [Autonomous Frontier-Based Exploration with VLM Guidance](https://arxiv.org/abs/2605.23165) — closest match, ground robot
- [Efficient Navigation in Unknown Indoor Environments with VLMs](https://arxiv.org/html/2510.04991v2)
- [AirHunt](https://arxiv.org/html/2601.12742) — aerial, handles VLM-latency-vs-flight-control timing
- [SoraNav](https://arxiv.org/abs/2510.25191) — aerial, geometric fallback when VLM output is bad
- [NavGPT](https://arxiv.org/abs/2305.16986), [VoroNav](https://arxiv.org/abs/2309.10329) — foundational VLM-as-planner papers

## Tech Stack (rough idea, subject to change as you design it)

Python, ROS 2 Humble, AirSim + PX4 SITL, MAVROS, ollama (LLaVA / LLaMA-Vision), OpenCV, numpy, matplotlib/seaborn for plotting results.
