System Architecture: VLM-as-Planner for UAV Navigation

Three loosely-coupled layers — perception, planning, execution — with clear ROS 2 interfaces between them.

── ENVIRONMENT ──────────────────────────────────────────
AirSim (Indoor Scene) + PX4 SITL (Flight Controller)
RGB-D camera · IMU · GPS · Depth map
↓ ROS 2 topics: /camera/rgb, /camera/depth, /mavros/local_position
── LAYER 1: PERCEPTION ──────────────────────────────────
Scene Describer
LLaVA / BLIP-2
RGB frame → natural language description
"A narrow corridor, door on the left, open space ahead"
Occupancy Mapper
Depth image → 3D voxel map
Tracks explored / unexplored regions
Built with OctoMap or simple grid
↓ scene description (string) + occupancy map (JSON/array)
── LAYER 2: PLANNING (Foundation Model) ─────────────────
VLM Exploration Planner
LLaMA-3-Vision (local) or GPT-4V (API)

Input prompt: Scene description + explored map + UAV pose + task ("explore fully")
Output: Next waypoint in relative coordinates + reasoning

"The open space ahead is unexplored. Move forward 3m and turn right to inspect the branching corridor."
↓ target waypoint (x, y, z, yaw)
── LAYER 3: EXECUTION ───────────────────────────────────
Safety Filter
Validates waypoint against occupancy map
Rejects if collision predicted
Falls back to hover if VLM fails
PX4 Waypoint Controller
MAVROS → PX4 OFFBOARD mode
Executes waypoint via position setpoints
Reports completion back to planner
↑ completion feedback loop back to Layer 1
Baseline comparison: Same setup but Layer 2 replaced with classical frontier exploration (geometric only) — measures how much semantic reasoning improves coverage efficiency.
