5-Month Project Timeline

Target: November 2026 master's application deadline. Each phase builds on the last and produces a concrete deliverable.

Phase 1 — Foundation & Environment Setup  ·  Weeks 1–3 (Jun 15 – Jul 6)
▸ Extend SITL project: add indoor AirSim scene (hospital / warehouse map)
▸ Verify RGB-D + depth topics flowing in ROS 2
▸ Run LLaVA / BLIP-2 locally on single frames — confirm inference works
▸ Build simple OctoMap or 2.5D grid occupancy mapper from depth stream
✦ Deliverable: drone hovers in AirSim, camera feed live, VLM describes scene in terminal
Phase 2 — Perception Layer  ·  Weeks 4–6 (Jul 7 – Jul 27)
▸ Build Scene Describer ROS 2 node: subscribes to /camera/rgb → calls LLaVA → publishes description
▸ Build Occupancy Mapper node: depth + pose → 2D explored/unexplored grid
▸ Tune description prompts for UAV-relevant output ("corridor", "open space", "obstacle")
▸ Log perception outputs for 5+ flight trajectories, verify quality
✦ Deliverable: perception pipeline running at ~2 Hz, scene descriptions logged to rosbag
Phase 3 — VLM Planning Layer  ·  Weeks 7–10 (Jul 28 – Aug 24)
▸ Build VLM Planner node: takes scene description + occupancy map → prompts LLM → parses waypoint
▸ Design and iterate on the planning prompt (few-shot examples, chain-of-thought)
▸ Build Safety Filter: validate waypoint against occupancy map before passing to PX4
▸ Close the loop: Planner → Safety Filter → MAVROS → PX4 → new observation → Planner
▸ Implement classical frontier exploration baseline (geometric only) for comparison
✦ Deliverable: drone autonomously explores room using VLM waypoints, baseline also running
Phase 4 — Evaluation & Analysis  ·  Weeks 11–14 (Aug 25 – Sep 21)
▸ Define metrics: coverage %, time-to-explore, collision rate, semantic accuracy
▸ Run 20+ trials each: VLM planner vs classical frontier vs random walk
▸ Ablation: local LLM vs API VLM, with/without scene description, prompt variants
▸ Generate plots, tables, failure case analysis
▸ Document findings — what did the VLM get right? Where did it fail?
✦ Deliverable: evaluation report with quantitative results and failure analysis
Phase 5 — Real-World Attempt  ·  Weeks 15–18 (Sep 22 – Oct 19)
▸ Port ROS 2 stack from SITL to physical drone (reuse skynet-ws setup)
▸ Replace AirSim camera topics with real sensor topics (same interface)
▸ Indoor flight tests in controlled environment — reduce speed, add extra safety limits
▸ Document sim-to-real gap: what broke, what transferred cleanly
✦ Deliverable: at least one successful real-world exploration run (or documented attempt + lessons learned)
Phase 6 — Write-Up & Portfolio  ·  Weeks 19–20 (Oct 20 – Nov 2)
▸ Write technical report / short paper (4–6 pages, NeurIPS/IROS format)
▸ Record demo video: simulation + real-world side by side
▸ Publish code on GitHub with README
▸ Draft resume bullet + SOP paragraph
✦ Deliverable: GitHub repo + demo video + technical report — ready for ETH Zurich application
⚠ Buffer: Phase 5 (real-world) is optional-but-valuable. If it slips, simulation results alone are sufficient for the application. The report and video are the non-negotiables.
